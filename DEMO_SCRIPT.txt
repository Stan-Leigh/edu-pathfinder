[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture of how a student's request is processed. (pause)
First, every request goes into the START node and immediately hits our Security Checkpoint.
The Security Checkpoint acts as a firewall, scrubbing PII, blocking prompt injections, and checking age consent.
If a security issue is detected, it routes immediately to the Security Event Handler, bypassing all sub-agents.
Otherwise, the clean query is sent to the Edu Orchestrator agent.
The Orchestrator utilizes AgentTools to delegate tasks to two specialized LLM sub-agents: the Curriculum Agent, which designs the personalized curriculum, and the Tutor Match Agent, which finds available free tutoring resources.
Both sub-agents make calls to our local Model Context Protocol server to fetch curriculum guides and verify available tutors.
Finally, the generated recommendation flows to the Human Approval node, pausing for user validation before completing.

[2:00 - LIVE DEMO]
(switch to playground at localhost:18081)
Let's look at a live demo of the agent in action. (pause)
We will send a standard request: "I am in Grade 10 and struggling with quadratic equations. Can you design a study plan and find a tutor?" (pause)
When we run this, the Security Checkpoint passes the clean query to the Orchestrator, which delegates to the sub-agents.
The Curriculum Agent fetches quadratic equation worksheets using our MCP tool, and the Tutor Match Agent looks up free math tutors.
The system then halts and prompts us: "Do you approve this study path? (yes/no)".
If I type "yes", the system moves to final_output, saving the finalized plan in state and activating the student dashboard.
(pause)
Now, let's see what happens if a student inputs sensitive personal data. If I type: "My name is John, student ID SID-99321. Email me at john@example.com. I need study help for Biology."
If we look at our logs, the email and student ID are immediately replaced with redaction placeholders before reaching the LLMs, keeping the student's identity completely safe.
(pause)
Lastly, if an underage student under 13 tries to use it without supervision, saying "I am 10 years old and need help", the Security Checkpoint flags the age consent check and stops the flow immediately with a warning, prompting for guardian approval.
